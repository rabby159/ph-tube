Discussing the scope of var, let, and const

var: The scope of a var variable is functional scope. variabale short form is "var". This is a keyword of JavaScript. Basically var keyword is container for storing data. var keyword is too old. Its used maximum time 1995 to 2015. Now, If I have a old browsers then I use it as well.

let: The scope of a let variable is block scope. let also a JavaSctipt keyword. let keyword introduced in ES6. let keyword can not be redeclared. But, if we need we can reassign let keyword. In modern javascript maximum developer use let. Its work all browsers as well as.

const: The scope of a const variable is block scope. const also a JavaSctipt keyword. const keyword introduced in ES6(2015). const keyword can not be redeclared also its can not be use reassigned. We use const keyword when we know stored value are not change avable.

The use cases of null and undefined

null: Null is a intentional absence assignment value. It can be assigned to a variable as a representation of no value. It is a Primitive value in Javascript. Its syntax is -: 'null'

undefined: Undefined means the variable has been declared, but its value has not been assigned. It is the global object. It is an ECMAScript1 (ES1) feature.

Representational State Transfer (REST)

In particular, Representational State Transfer (REST) is a software architectural style that developers apply to web APIs. REST APIs provide simple, uniform interfaces because they can be used to make data, content, algorithms, media, and other digital resources available through web URLs. Essentially, REST APIs are the most common APIs used across the web today.
